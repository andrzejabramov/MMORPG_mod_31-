## 1. Обзор проекта

**MMORPG Fan Board** — это доска объявлений для фанатского сервера MMORPG. Платформа позволяет пользователям регистрироваться, создавать объявления с форматированным текстом (WYSIWYG), оставлять отклики и управлять ими. Реализована система уведомлений по email и новостные рассылки.

### Функциональные требования (из ТЗ)

| № | Требование | Реализация |
|---|------------|------------|
| 1 | Регистрация по email с подтверждением | ✅ |
| 2 | Создание/редактирование объявлений (WYSIWYG) | ✅ |
| 3 | Категории объявлений (10 фиксированных) | ✅ |
| 4 | Отклики на объявления (простой текст) | ✅ |
| 5 | Уведомления автору о новом отклике | ✅ |
| 6 | Уведомления автору отклика о принятии | ✅ |
| 7 | Приватная страница с откликами + фильтрация | ✅ |
| 8 | Принятие/удаление откликов | ✅ |
| 9 | Новостные рассылки (кастомные команды) | ✅ |
| 10 | Система прав доступа | ✅ |

Этап 3: Регистрация и аутентификация
Механизм работы:

Пользователь вводит email и пароль

Создается неактивный пользователь (is_active=False)

Генерируется 6-значный код (OneTimeCode)

Отправляется email с кодом (сигнал send_verification_code)

Пользователь вводит код на странице /accounts/verify/

При успехе: is_active=True, is_verified=True, автоматический вход

Сигнал 1: Уведомление автору о новом отклике

@receiver(post_save, sender=Response)
def notify_author_on_new_response(sender, instance, created, **kwargs):
    if created and instance.post.author != instance.author:
        send_mail(
            subject=f'Новый отклик на {instance.post.title}',
            message=f'Пользователь {instance.author.username} оставил отклик...',
            from_email=settings.DEFAULT_FROM_EMAIL,
            recipient_list=[instance.post.author.email]
        )

Этап 6: Новостные рассылки (кастомные команды)

Файл: accounts/management/commands/send_newsletter.py

# Тестовая отправка
python manage.py send_test_email --to-admin

# Новостная рассылка всем подтвержденным пользователям
python manage.py send_newsletter \
  --subject "Новости сервера" \
  --message "Текст рассылки" \
  --recipients verified

# Режим проверки (dry run)
python manage.py send_newsletter --subject "Тест" --message "Тест" --dry-run

Параметры команды:

--subject — тема письма

--message — текст письма (или --file для загрузки из файла)

--recipients — all, active, verified, test

--batch-size — размер партии (по умолчанию 50)

--dry-run — только показать получателей
